# Security Policy

## Scope

This repository contains the PoG Framework documents (specification, threat model, conformance plan, claims policy). It contains no product code.

Security findings can nevertheless concern:

- **Framework-level weaknesses**: a flaw in the specified sealing, verification, revocation or tenant-attribution model that would allow a conformant implementation to be attacked. These are the most valuable reports.
- **Reference implementation (PREVORN)**: vulnerabilities in the PREVORN platform itself.

## Reporting

- Framework-level findings: open a public issue if the discussion can be public, or email spec@pog-protocol.org if you believe publication should wait.
- PREVORN platform vulnerabilities: email security@prevorn.ch. Do not open public issues for exploitable product vulnerabilities.

We acknowledge reports within 5 business days.

## Commitments

- No legal action against good-faith research conducted without service disruption or access to third-party data.
- Documented framework flaws are answered publicly: accepted, mitigated, or rebutted with reasons. Silence is not an option we allow ourselves.
- Reporters are credited in the changelog unless they prefer otherwise.
